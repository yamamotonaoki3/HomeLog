## データベース
- PostgreSQL
- 理由: JSON型・配列型など柔軟なデータ型、MVCCによる整合性の強さ、割り勘精算・世帯権限のような複雑なリレーションとの相性の良さから採用（MySQLではなくPostgreSQLを選定）

## インフラ・IaC
- クラウド: AWS
- IaC: Terraform でAWSリソース一式を管理
- 想定リソース構成（詳細はインフラ設計時に確定）
  - コンピュート: ECS Fargate または EC2（Spring Boot稼働）
  - DB: RDS for PostgreSQL（無料利用枠 db.t3.micro 想定）
  - フロント配信: S3 + CloudFront
  - ファイルストレージ: S3（レシピ画像・手書き画像等）
  - 認証: Spring Security + jjwt によるアプリ内JWT発行（Cognito等のマネージド認証は使用しない）

## 運用方針
- 初期フェーズ（家族利用）: AWS無料利用枠を最大限活用し、運用コストを抑える
- 将来の全国展開フェーズ: 利用規模に応じてインフラ構成（ECS/RDSのスケールアップ、マルチAZ化等）を再設計する
