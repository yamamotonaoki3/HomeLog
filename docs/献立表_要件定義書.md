# 献立表 要件定義書

## 1. 概要
レシピの登録・管理と、日単位での献立の計画を行う機能（[[構想.md]] 3-3章参照）。在庫管理・家計簿との自動連携は行わない独立機能。

## 2. 権限
- レシピ・献立表の登録・編集は世帯メンバー全員が自由に行える（権限制限なし）。

## 3. レシピ登録

### 3-1. 手動登録
- 材料・分量・手順等を入力してレシピを登録できる。

### 3-2. 手書きレシピの画像解析登録
- 手書きレシピの写真をアップロードすると、画像解析により材料・分量・手順を自動構造化する。
- 解析結果はユーザー確認フォームに一時反映され、ユーザーが内容を確認・編集したうえでレシピとして登録する（自動解析のみで即登録はしない）。

### 3-3. WEBレシピの登録
- レシピのURLを貼り付けることで登録できる。
- 保存するのはURL・タイトル・サムネイル画像のみとし、詳細（材料・手順など）はアプリ内には保持せず、リンク先の元ページを参照する形とする。
- WEBレシピ登録時、ユーザー独自のメモ（例：「塩を足す」等）を残せる。

### 3-4. お気に入り
- 登録済みのレシピ（手動登録・WEBレシピ問わず）をお気に入り登録できる。

## 4. 献立表
- 1日単位でレシピを登録する（朝・昼・晩などの区分は設けない）。
- 登録済みレシピ・お気に入りレシピ・WEBレシピのいずれも献立に登録できる。
- 表示は「日表示」と「週表示（カレンダー形式で1週間分を一覧表示）」を切り替えられるようにする。登録はどちらの表示からでも1日単位で行う。

## 5. データ連携方針（構想書より）
- 在庫管理とは連携しない（材料個数の自動消費は行わない。手持ちの食材で作る前提のため）。
- 家計簿とは連携しない。

## 6. 今後の検討事項
- WEBレシピのURL貼り付け時のタイトル・サムネイル自動取得方法（著作権・利用規約上の扱い含む、技術スタック検討時に詳細を詰める）
- 手書きレシピ画像解析の実現方式（OCR/AI画像解析サービスの選定は技術スタック検討時に決定）
- 週単位での献立表示・コピー機能の要否（構想書では日単位・週単位の両方に言及があったため、週単位はカレンダー表示のみか登録単位も含むか要確認）
