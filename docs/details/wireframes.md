# ワイヤーフレーム

[← 要件定義書に戻る](../requirements.md)

S-01〜S-18の全画面について、要件定義段階の骨組みレベルでワイヤーフレームをまとめる。実際のUIデザイン（配色・フォント・詳細CSS）の決定前段階の「叩き台」として、ASCIIアート風のレイアウト図で画面内の要素配置・サイズ感を示し、補足としてMermaidのブロック図（要素の関係性）も併記する。各画面の項目は対応する[機能別要件定義書](../requirements.md#5-機能要件)の内容に基づく。

## S-04 トップ画面（生活ダッシュボード＋カレンダー）

対応: [F06_kakeibo_event](features/F06_kakeibo_event.md), [F12_kakeibo_household_summary](features/F12_kakeibo_household_summary.md), [F07_zaiko_inventory](features/F07_zaiko_inventory.md), [F08_zaiko_shoppinglist](features/F08_zaiko_shoppinglist.md), [F10_kondate_menu](features/F10_kondate_menu.md)

確認系の情報（今日の状況・今月のお金・買い物/在庫）を画面遷移なしで把握できる「生活ダッシュボード」として再定義する（[common-notes.md](common-notes.md) 9章の方針）。旧S-16（世帯合計支出画面）・旧S-20（イベント別支出サマリーモーダル）はここに統合し廃止した。

┌──────────────────────────────────────────────┐
│ [家計簿] [在庫管理] [献立表]              🔔2  │
├──────────────────────────────────────────────┤
│ ■ 今日の状況（7/10）                           │
│  収支: -500円   献立: カレー系(ラフ)            │
│  イベント: 📌学習                               │

│  イベント別支出（今年）: 父の日3,000円 推し活12,000円│
├──────────────────────────────────────────────┤
│ ■ 買い物・在庫                                  │
│  買い物リスト: 3件    在庫不足: 2件              │
│  よく使う品目: 卵・牛乳・醤油                    │

│ 日 │ 月 │ 火 │ 水 │ 木 │ 金 │ 土             │

│    │    │    │    │📌学習│    │  🎉父の日      │
│ -1,200円│ -300円│ +0円│ +0円│-500円│ +0円│-3,000円  │

│   📌=繰り返しイベント。日付クリックで日次詳細）  │

```mermaid
flowchart TD
    subgraph Screen["S-04 トップ画面（生活ダッシュボード）"]
        direction TB
        Nav["ナビゲーション（家計簿 / 在庫管理 / 献立表）＋通知バッジ"]
        Today["「今日の状況」カード：収支・献立・イベント"]
        Money["「今月のお金」カード：個人支出・世帯合計対象額・\n未精算件数/金額・イベント別支出サマリー（当年合計）"]
        Stock["「買い物・在庫」カード：買い物リスト件数・在庫不足件数・よく使う品目"]
        Header["◀ 2026年7月 ▶"]
        Calendar["月間カレンダー\n（各日付セルに献立・固定費支払日・イベント名・\nその日の収支を表示。繰り返しイベントはアイコンで区別）"]
    end
    Nav --- Today --- Money --- Stock --- Header --- Calendar
    Calendar -- 日付をクリック --> DayModal(["S-19 日次詳細モーダル"])
    Money -- 「精算一覧を見る」リンク --> S07(["S-07 割り勘精算一覧画面"])
    Stock -- 「買い物リストを見る」リンク --> S10(["S-10 在庫・買い物リスト画面"])
```

- カレンダーの各日付セルの具体的な情報密度・アイコンデザインは未定。今後の詳細検討事項とする。
- 通知バッジは、通知有効な繰り返しイベントの当日発生件数を表示するイメージ。
- 「今月のお金」カード内のイベント別支出サマリーのみ、他の項目（今月）と異なり**当年（1/1〜12/31）の合計金額**を表示する。そのイベントに今年いくら使ったかを確認する目的のため。
- イベント別支出サマリーは、件数が多い場合は上位数件＋「もっと見る」等の表示に将来対応する（今後の検討事項）。
- 各カードの数値・リンク先は、対応する各機能画面の情報をそのまま参照する（新たな計算ロジックは持たない）。

## S-10 在庫・買い物リスト画面

対応: [F07_zaiko_inventory](features/F07_zaiko_inventory.md), [F08_zaiko_shoppinglist](features/F08_zaiko_shoppinglist.md)

在庫管理と買い物リストは密連携するため、PCでは左右パネルで同時表示し、画面遷移せずに両方確認できるようにする（[common-notes.md](common-notes.md) 9章）。モバイル幅ではタブ切り替えにフォールバックする。

```text
┌──────────────────────────────┬───────────────────────────────┐
│ 在庫一覧          [ 在庫を登録 ]│ 買い物リスト  [ 品目を手動追加 ] │
│ カテゴリー/店舗マスタ管理        │ 並び替え: [あいうえお順▼]        │
├────────┬──────┬──────────┬────┤├────────┬──────┬───────────┬───┤
│ 品名    │カテゴリ│在庫個数    │閾値 ││ 品名    │購入済│購入個数     │削除│
├────────┼──────┼──────────┼────┤├────────┼──────┼───────────┼───┤
│ 牛乳    │乳製品 │[－]1.0[＋]│0.5 ││ 卵      │ □   │[－] 0 [＋] │ ✕ │
│ 卵      │卵     │[－]0.0[＋]│1.0 ││ 牛乳    │ ☑   │[－] 1 [＋] │ ✕ │
│        │      │(1/0.1切替) │    │├────────┴──────┴───────────┴───┤
│        │      │            │    ││          [ 更新 ]              │
└────────┴──────┴──────────┴────┘└───────────────────────────────┘
   （モバイル幅では [在庫一覧] [買い物リスト] タブに切り替え表示）
```

```mermaid
flowchart TD
    subgraph Screen["S-10 在庫・買い物リスト画面"]
        direction LR
        subgraph InventoryPanel["在庫一覧パネル"]
            direction TB
            AddBtn["「在庫を登録」ボタン"]
            MasterLink["カテゴリー/店舗マスタ管理リンク"]
            InvList["在庫一覧テーブル（各行に＋/－ボタン、\n増減ステップ1/0.1切り替えチップ付き）"]
        end
        subgraph ShoppingPanel["買い物リストパネル"]
            direction TB
            Sort["並び替え：あいうえお順 / カテゴリー順 / 買う店順"]
            ShopAddBtn["「品目を手動追加」ボタン"]
            ShopList["買い物リストテーブル（購入個数は＋/－ボタンで1個単位に増減）"]
            UpdateBtn["「更新」ボタン（購入分を在庫に一括反映）"]
        end
    end
    AddBtn --- MasterLink --- InvList
    Sort --- ShopAddBtn --- ShopList --- UpdateBtn
    InvList -. "PC:左右同時表示 / モバイル:タブ切替" .- ShopList
```

在庫一覧テーブルの列例：

| 品名 | カテゴリー | 買う店 | 在庫個数（－/＋ボタン、ステップ1 or 0.1切り替え） | 閾値 |
| --- | --- | --- | --- | --- |

買い物リストテーブルの列例：

| 品名 | カテゴリー | 買う店 | 購入済みチェック | 購入個数（－/＋ボタン、1個単位） | 手動削除ボタン |
