# F-02 世帯グループ管理

[← 要件定義書に戻る](../../requirements.md)

---

## 1. 概要

ユーザーが「世帯グループ」を作成・参加し、精算・在庫・献立を世帯単位で共有できるようにする（[common-notes.md](../common-notes.md) 1章参照）。個人の家計簿は世帯グループに関わらず個人管理のまま。

**MVPでは1ユーザーにつき1世帯グループのみに所属可能とする**（複数世帯への同時所属は不可）。既に世帯グループに所属しているユーザーが別の世帯を作成・参加しようとした場合はエラーとする。将来的に複数世帯への所属が必要になった場合は`household_members`テーブル構造はそのまま多対多に対応できるため、アプリケーション層の制約を緩和するだけで拡張可能。

## 3. 業務フロー

```mermaid
flowchart TD
    A([ユーザー登録完了]) --> B{世帯グループ}
    B -- 新規作成 --> C[世帯グループ名を入力]
    C --> D[households テーブルに保存\nhousehold_members に自分を登録]
    B -- 既存に参加 --> E[招待コード/招待リンクを入力]
    E --> F{招待コード有効?}
    F -- NG --> G[エラーメッセージ]
    G --> E
    F -- OK --> H[household_members に自分を追加]
    D --> I[S-04 トップ画面へ]
    H --> I
```

## 4. IPO

### 世帯グループ作成

| 項目 | 内容 |
| --- | --- |
| 入力 | 世帯グループ名 |
| 処理 | 既にいずれかの世帯グループに所属していないか確認 → households テーブルに保存（`invite_code`を自動生成） → household_members に作成者を登録 |
| 出力 | 作成した世帯グループ（招待コードを含む） / エラーメッセージ |

### 世帯グループ参加

| 項目 | 内容 |
| --- | --- |
| 入力 | 招待コード |
| 処理 | 既にいずれかの世帯グループに所属していないか確認 → 招待コードの有効性確認（[data-model.md](../data-model.md)の`households.invite_code`と照合） → household_members に自分を追加 |
| 出力 | 参加完了 / エラーメッセージ |
