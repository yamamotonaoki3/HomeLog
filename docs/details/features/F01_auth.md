# F-01 認証（ユーザー登録・ログイン）

[← 要件定義書に戻る](../../requirements.md)

---

## 1. 概要

メールアドレス・パスワードによるユーザー登録とログイン。JWTによる認証を行う（[技術スタック.md](../../技術スタック.md)参照）。

## 2. 対象画面

| 画面ID | 画面名 |
| --- | --- |
| S-01 | ログイン画面 |
| S-02 | ユーザー登録画面 |
| S-17 | パスワードリセット画面 |

## 3. 業務フロー

```mermaid
flowchart TD
    A([開始]) --> B{アカウントあり?}
    B -- なし --> C[S-02 ユーザー登録画面\nメール・パスワード・表示名を入力]
    C --> D[users テーブルに保存\nパスワードはBCryptでハッシュ化]
    D --> E[S-03 世帯グループ作成/参加画面へ]
    B -- あり --> F[S-01 ログイン画面\nメール・パスワードを入力]
    F --> G{認証OK?}
    G -- NG --> H[エラーメッセージ]
    H --> F
    G -- OK --> I[JWT発行]
    I --> J[S-04 トップ画面へ]
    F -- 「パスワードを忘れた場合」リンク --> K[S-17 パスワードリセット画面]
```

## 3-1. パスワードリセットフロー

```mermaid
flowchart TD
    A([S-17 パスワードリセット画面]) --> B[メールアドレスを入力]
    B --> C[リセット用トークンを発行し\nメールでリセットリンクを送信]
    C --> D[ユーザーがメール内リンクをクリック]
    D --> E{トークン有効?}
    E -- 無効/期限切れ --> F[エラーメッセージ\n再度リセット申請へ]
    E -- 有効 --> G[新しいパスワードを入力]
    G --> H[users.password_hash を更新]
    H --> I[S-01 ログイン画面へ]
```

## 4. IPO

### ユーザー登録

| 項目 | 内容 |
| --- | --- |
| 入力 | メールアドレス・パスワード・表示名 |
| 処理 | 入力チェック → メール重複チェック → パスワードハッシュ化 → users テーブルに保存 |
| 出力 | 登録完了 / エラーメッセージ |

### ログイン

| 項目 | 内容 |
| --- | --- |
| 入力 | メールアドレス・パスワード |
| 処理 | ユーザー検索 → パスワード照合 → JWT発行 |
| 出力 | アクセストークン・リフレッシュトークン / エラーメッセージ |

### パスワードリセット申請

| 項目 | 内容 |
| --- | --- |
| 入力 | メールアドレス |
| 処理 | ユーザー検索 → リセットトークン発行（有効期限つき） → メール送信 |
| 出力 | 送信完了メッセージ（メール存在有無に関わらず同一メッセージを返し、メールアドレスの存在を推測されないようにする） |

### パスワードリセット実行

| 項目 | 内容 |
| --- | --- |
| 入力 | リセットトークン・新しいパスワード |
| 処理 | トークンの有効性・期限チェック → users.password_hash を更新 → トークンを無効化 |
| 出力 | リセット完了 / エラーメッセージ |

## 5. 入力チェック仕様

| 項目 | 必須 | 形式・制約 |
| --- | --- | --- |
| メールアドレス | ○ | メール形式、UNIQUE |
| パスワード | ○ | 8文字以上（詳細な強度要件は今後検討） |
| 表示名 | ○ | 1〜50文字 |
