# HomeLog 全体構想書

## 1. アプリの目的
個人の財政管理を行いながら、1世帯内で協力して購入したものの精算・食材等の在庫管理を一括で行えるアプリ。

## 2. 想定ユーザー像
- 単位は「個人」。ただし「世帯グループ」を作成し、世帯内での精算・共有管理を行う。
- 個人財政は個人が管理し、世帯全体としての合算財政は管理しない（世帯としての家計簿は持たない）。
- 世帯管理では、協力購入した物品やお金の貸し借りの精算関係を管理する。
- シェアハウスのような、家族に限らない複数人同居でも利用できることを想定。
- 将来的に自分・家族以外の第三者にも公開する可能性がある。

## 3. 各機能の概要

### 3-1. 家計簿
- 個人の支出管理：日時、金額、使用用途、カテゴリー、支払った人、メモを登録可能
- 世帯で割り勘したものの精算関係の管理
- 世帯に関係のない相手（世帯外の人）との割り勘・精算管理も可能にしたい
- 固定費（水道・光熱費・家賃・ネット代・携帯料金等）を毎月の支払日とともに登録できる
- 「イベント」と「支出」をセットで登録できる（例：サークル会費1,000円）
- カテゴリー別の支出検索
- イベントごとの使用金額をTOP画面等ですぐ確認できる機能（例：父の日、推し活などの用途ごとに金額を集計表示）

### 3-2. 食材の在庫管理
- 在庫管理項目：品名、カテゴリー、買う店（任意）、個数（小数点第一位まで入力可）、買い物リストへの追加閾値（個数）を登録
- 在庫が閾値を下回ったら自動的に買い物リストへ追加
- 買い物リストの並び替え：あいうえお順／カテゴリー順／買う店順
- 買い物リストでの購入チェックと購入個数の入力
- 「更新」ボタンで一括して在庫管理表へ加算し、閾値条件を外れた項目は買い物リストから自動除外

### 3-3. 献立表
- レシピ登録（手書きレシピを画像解析でテキスト化して登録する機能を含む）
- WEB上のレシピをURL貼り付けで引用登録
- お気に入りレシピ登録
- WEBレシピ登録時に自分用メモ（例：「塩を足す」等）を残せる
- 登録済みレシピ／WEBレシピを日単位・週単位の献立として登録

## 4. 機能間の連携方針
- **献立表 ⇔ 在庫管理**：連携なし。献立はレシピを参考に手持ちの食材で作るため、材料個数と在庫の自動連携は行わない。
- **在庫管理 ⇔ 買い物リスト**：密連携。在庫が閾値を下回ると自動的に買い物リストに追加され、購入更新時に在庫へ反映される（3-2参照）。
- **買い物リスト ⇔ 家計簿**：連携なし。買い物リスト以外の支出も多いため、自動連携はしない。

## 5. データ構造の方針
- 在庫管理と買い物リストは共通のデータ（同一テーブル/密連携）として扱う。
- 家計簿・献立表は、在庫管理／買い物リストとは別個の独立したテーブル群として設計する。

## 6. マルチユーザー・権限に関する考慮点
- 世帯グループ単位でユーザーを管理するマルチユーザー構成。
- 個人の家計簿・在庫・献立データは本人のみが編集可能（他の世帯メンバーは閲覧・編集不可）。
- 割り勘・立て替えの精算は、支払いを受ける側（相手）が承認しないと「精算済み」にならない（承認フロー必須）。
- 在庫管理・献立登録には権限制限を設けず、世帯メンバーであれば誰でも自由に操作可能。
- データ分離（他世帯のデータに一切アクセスできないようにするアクセス制御設計）は、技術スタック検討時に詳細を詰める。
