# 家計簿 要件定義書

## 1. 概要
個人の支出管理を軸に、世帯内・世帯外との割り勘精算、固定費の自動計上、イベント単位の支出集計を行う機能（[[構想.md]] 3-1章参照）。世帯全体の合算家計簿は持たず、あくまで個人の家計簿として扱う。

## 2. 権限
- 個人の支出データは本人のみが編集可能（他の世帯メンバーは閲覧・編集不可、[[構想.md]] 6章）。
- 割り勘精算は、相手側の承認（受領確認）があって初めて「精算済み」となる。

## 3. 個人支出管理

### 3-1. 登録項目
| 項目 | 必須/任意 | 内容 |
|---|---|---|
| 日時 | 必須 | 支出発生日 |
| 金額 | 必須 | 支出金額 |
| 使用用途 | 必須 | 何に使ったか |
| カテゴリー | 必須 | カテゴリーマスタから選択 |
| 支払った人 | 必須 | 誰が支払ったか（世帯メンバーの中から選択） |
| メモ | 任意 | 自由記述 |

### 3-2. カテゴリーマスタ
- 家計簿専用のカテゴリーマスタを持つ（在庫管理の食材カテゴリーとは別個。例：食費、日用品、交際費、光熱費、住居費、通信費 等）。

### 3-3. 検索・集計
- カテゴリー別に支出を検索できる。

## 4. 割り勘・精算管理

### 4-1. 精算の基本フロー
- 支出登録時に割り勘対象者を選び、各自の負担額を自動計算する。
- 立て替えられた側が支払うタイミングで支払い、立て替えた側が「受領」ボタンを押すことで精算済みとなる。精算履歴として残る。
- 逆に、立て替えた側から「請求」を行うこともできる。請求した履歴も残り、相手から支払われたら「受領」ボタンを押して精算済みとする。

### 4-2. 世帯外の相手との精算
- 相手がアプリ利用者の場合：世帯メンバーとの精算と同じ承認フロー（相手が受領確認）を適用する。
- 相手が非利用者の場合：自己申告のみで精算完了として扱う（相手側の承認は発生しない）。

### 4-3. 請求の保留
- 請求に対して相手が即座に応じられない場合のために「保留中」状態を用意する。
- 保留中の請求にはコメントを残してやり取りできる（例：相手からの「来週払う」等の返信）。

## 5. 固定費管理
- 以下をデフォルトの固定費項目として用意し、金額と支払日を入力するだけで登録できるようにする：家賃、水道代、電気代、ガス代、インターネット代、携帯電話代、サブスクリプション（動画・音楽配信等）
- デフォルトに含まれない固定費は、ユーザーが任意に項目を追加登録できる。
- 登録された固定費（金額・支払日）は、毎月自動的に支出として計上される。
- 固定費の金額を変更した場合、変更した月から新しい金額で計上される（当月分が未計上なら当月から、計上済みなら翌月から新金額を適用）。

## 6. イベント管理
- イベント（例：父の日、推し活、旅行など）を先に作成し、複数の支出を後から紐付けられる。
- イベントに紐付いた支出の合計金額を、TOP画面等ですぐに確認できる（用途：金額の一覧表示）。

## 7. データ連携方針（構想書より）
- 買い物リストとは連携しない（買い物リスト以外の支出も多いため、自動連携はしない）。買い物での購入も含め、支出は都度手動で家計簿に登録する。
- 在庫管理・献立表とは連携しない。

## 8. 通貨単位
- 支出データの通貨単位は日本円固定とする（多通貨対応は行わない）。
