# 在庫管理 要件定義書

## 1. 概要
世帯内の食材等の在庫を一元管理し、在庫が一定数を下回った品目を自動的に買い物リストへ連携する機能。買い物リストで購入を記録すると、在庫数に自動反映される（[[構想.md]] 3-2, 4章参照）。

## 2. 権限
- 在庫の登録・編集・削除は世帯メンバー全員が自由に行える（権限制限なし）。

## 3. マスタデータ

### 3-1. カテゴリーマスタ
- システムデフォルトのカテゴリーとして以下9つを用意する：野菜、肉、魚介、乳製品、卵、調味料、飲料、冷凍食品、乾物、その他
- ユーザーは任意にカテゴリーを追加できる。
- ユーザーが追加したカテゴリーは編集・削除が可能（デフォルトカテゴリーは編集不可、または要検討）。
- カテゴリーは世帯単位で共有される。
- 使用中（在庫アイテムに紐付いている）のカテゴリーは削除不可とする。削除するには先に対象アイテムのカテゴリーを付け替える必要がある。

### 3-2. 店舗マスタ
- 「買う店」はテキスト自由入力ではなく、店舗マスタから選択する形式。
- 店舗マスタはユーザーが任意に登録・編集・削除できる。
- 店舗マスタは世帯単位で共有される。
- 使用中（在庫アイテムに紐付いている）の店舗は削除不可とする。削除するには先に対象アイテムの店舗を付け替えるか、店舗指定を解除する必要がある。

## 4. 在庫アイテムの登録項目
| 項目 | 必須/任意 | 内容 |
|---|---|---|
| 品名 | 必須 | 食材名 |
| カテゴリー | 必須 | カテゴリーマスタから選択 |
| 買う店 | 任意 | 店舗マスタから選択 |
| 在庫個数 | 必須 | 小数点第一位まで入力可（例：1.5） |
| 買い物リスト追加閾値 | 必須 | この個数を下回ったら自動で買い物リストに追加 |

## 5. 在庫管理の挙動
- 在庫個数が閾値を下回った場合、自動的に買い物リストへ追加される。
- 在庫が0になっても品目は在庫一覧から削除されず、「在庫0」として表示され続ける。
- 在庫個数は小数点第一位まで管理する。
- 在庫個数は0未満にならないよう入力を制限する（購入個数の入力ミス等でマイナスになる更新は許可しない）。

## 6. 買い物リスト

### 6-1. 追加方法
- 自動追加：在庫が閾値を下回った品目が自動的に追加される。
- 手動追加：閾値を超えていても、ユーザーが任意の品目を手動で買い物リストに追加できる。
- 手動削除：自動・手動を問わず、ユーザーは買い物リストから任意の品目を手動で削除できる。

### 6-2. 並び替え
- あいうえお順
- カテゴリー順
- 買う店順

### 6-3. 購入処理
- 各品目に「購入済みチェック」と「購入個数」を入力できる。
- 「更新」ボタン押下で、チェックの入った品目を一括で在庫管理表へ加算する。
- 加算後、在庫個数が閾値を上回った品目は自動的に買い物リストから除外される（手動追加分は閾値に関わらず購入処理により除外）。

## 7. データ連携方針（構想書より）
- 在庫管理と買い物リストは密連携（同一データソース）。
- 献立表とは連携しない（材料個数の自動消費は行わない）。
- 家計簿とは連携しない（買い物リストでの購入と家計簿の支出登録は別操作）。
